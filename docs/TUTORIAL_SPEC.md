# チュートリアル機能 実装要件定義書（AI実装用）

このドキュメントは、本アプリケーション（HDL-Sim）に「初回起動時チュートリアル機能」および「Helpからのチュートリアル表示機能」を実装するAIモデルに向けた要件定義および実装指示書です。
実装を担当するAIは、以下の要件を満たすようにコードを記述・修正してください。

## 1. 実装の目的
ユーザーがツールを初めて開いた際に、ツールの使い方やエラー発生時の対処法を分かりやすく案内し、スムーズに利用を開始できるようにします。また、一度閉じた後でも必要に応じて再確認できる導線を用意します。

## 2. 機能要件

### 2.1 初回起動時の自動表示
- アプリケーションの初回起動時のみ、自動的にチュートリアル（モーダルダイアログ、または専用の案内画面）を表示してください。
- ユーザーがチュートリアルを閲覧した（またはスキップした）ことを記録するため、設定ファイルやローカルストレージ等の永続化領域に `has_seen_tutorial` (または同等の) フラグを保存し、次回以降は自動表示しないように制御してください。

### 2.2 Helpからの再表示機能
- アプリケーションのメニューバーや設定項目などに「Help（ヘルプ）」メニューを追加、あるいは既存のメニューを利用して、「チュートリアルを表示」するボタン/リンクを実装してください。
- これにより、ユーザーは任意のタイミングでチュートリアルを再確認できるようにします。

## 3. チュートリアルのコンテンツ要件

チュートリアルの内容は、複数ステップ（ページ送り）などを活用し、以下の内容を**必ず**含めてください。

### Step 1: ツールの概要
- 本ツール（HDL-Sim）がどのような目的のツールであるか、何ができるシミュレータであるかの簡単な概要を説明してください。

### Step 2: プロジェクトの作り方とRUN（実行）の仕方
- ユーザーインターフェース（UX）に沿って、基本的な操作フローを説明してください。
- **記載すべき操作例:**
  1. プロジェクト（またはシミュレーションファイル）の新規作成・読み込み方法
  2. シミュレーションの実行（RUN）方法
  3. 結果の確認方法

### Step 3: エラー発生時の対応と問い合わせ先（重要）
- 本ツールは開発中であり、未実装の機能によるエラーや予期せぬ不具合が発生する可能性がある旨をユーザーに伝えてください。
- トラブルシューティングのフローとして、**以下の文面（意味合い）を必ず含めてください。**
  > 「エラーが発生した場合は、まずはエラー内容をご自身がお使いのAIアシスタントに質問して解決策を探ってみてください。」
  > 「AIに聞いても解決しない場合や、シミュレータ自体の不具合（バグ）であると思われる場合は、以下のメールアドレスまでご連絡ください。」
  > **連絡先:** `t230g108@gunma-u.ac.jp`

## 4. UI/UX の設計方針
- 長いテキストが一度に表示されるのを避け、ユーザーが読みやすいように箇条書きやステップ形式（「次へ」「前へ」ボタンでの切り替え等）を採用してください。
- 「閉じる」または「スキップ」ボタンを常設し、ユーザーがいつでもチュートリアルを抜けられるように配慮してください。
- 必要に応じて、実際の画面のスクリーンショットやアイコンが入るプレースホルダを設けておき、後から画像を差し替えやすい構造にしてください。
