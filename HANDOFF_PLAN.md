# UI・保存アーキテクチャ改修計画 (引き継ぎ資料)

現在の Web UI 独自の保存形式（`projects` フォルダへの JSON 保存）を廃止し、本来の `.spj` プロジェクト形式と生ソースコード (`.v`) を主軸とした分かりやすいディレクトリ構造へと改修します。

## 概要

1. **エディタUIの改修**
   - ウィンドウコントロール（最小化ボタンの左横付近）に、現在開いている単一の Verilog ソースコードを保存するための「`.v` 保存ボタン」を追加します。
2. **`Ctrl+S` の挙動変更 (.spj保存の連携)**
   - `Ctrl+S` を押した際、またはメニューから保存した際に `.spj` プロジェクトとして保存します。
   - さらに、保存された `.spj` 名に紐づく形で `verilog_sources/[spj名]/` というフォルダを自動生成し、そこに含まれるすべての `.v` ファイルを生テキストとして自動で書き出します。
3. **`File -> Open` の変更**
   - 従来の `projects` ではなく、`verilog_sources` フォルダ（または `.spj` の一覧）を優先して開く・探せるように API と UI の参照先を変更します。
4. **チュートリアルの更新**
   - `ui/js/tutorial.js` 等のチュートリアルテキストを更新し、`Ctrl+S` がプロジェクト全体（.spj）保存であること、タブ横のボタンが単一ファイル（.v）保存であることを明記します。

## 決定事項 (User Approved)
- **「単一 .v の保存」の挙動**: ブラウザのダウンロードダイアログではなく、バックエンドの `verilog_sources/[spj名]/` などの正しいパスに対して裏側で **直接上書き保存** する形式とします。（エディタとして自然な挙動）
- **チュートリアルへの反映**: 操作方法の変更に伴い、チュートリアルに説明を追加します。

## Proposed Changes

### UI / Frontend
#### [MODIFY] `ui/index.html`
- エディタのタブバー右上（最小化ボタン `_` の横）に `.v` 保存用のアイコンボタン (`<i class="fas fa-save"></i>` 等) を追加。

#### [MODIFY] `ui/js/app.js` & `ui/js/menu-bar.js`
- `Ctrl+S` のイベントリスナーを整理し、`.spj` のメタデータ保存と同時に、各エディタタブ内のコードを `verilog_sources` に送信するロジックへ変更。
- 新設した `.v` 保存ボタンのクリックイベントを追加（現在アクティブなファイルだけを上書き保存するAPIを叩く）。

#### [MODIFY] `ui/js/tutorial.js`
- 保存に関するチュートリアルステップを修正し、`.spj` 保存と `.v` 単体保存の違いを説明。

### Backend / API
#### [MODIFY] `src/hdl_sim/web/spj_store.py` (または新設)
- `save_spj_file()` の処理を拡張。`.spj` を `spj/` フォルダに保存するのと同時に、Payload に含まれるファイル群を `verilog_sources/[spjファイル名(拡張子なし)]/` 配下に `[ファイル名].v` として出力する機能を追加。

#### [MODIFY] `src/hdl_sim/web/app.py`
- 単一ファイルの上書き保存用APIを追加（または既存を修正）。
- `File -> Open` 用に参照される API の対象を `verilog_sources` や `spj` を基点にするよう変更。

## Verification Plan
1. エディタで適当なコードを書き、`Ctrl+S` を押す。
2. `spj/` フォルダに `.spj` が保存され、同時に `verilog_sources/[プロジェクト名]/` 内に対応する `.v` ファイルが生成・更新されていることを確認する。
3. 追加した `.v` 保存ボタンを押し、単一ファイルのみが `verilog_sources` 内に正しく上書き保存されることを確認する。
4. `File -> Open` から `verilog_sources` のファイルが正しく開けることを確認する。
5. チュートリアルを開き、新しい保存操作の説明が追加されているか確認する。
