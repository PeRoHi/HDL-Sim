HDL-Sim のアンインストール
==========================

通常
----
設定 → アプリ → インストールされているアプリ → 「HDL-Sim」→ アンインストール

スマート アプリ コントロールで unins000.exe がブロックされた場合
----------------------------------------------------------------
Windows 11 の「スマート アプリ コントロール」は、発行元が確認できない
unins000.exe（Inno Setup の削除プログラム）をブロックすることがあります。
HDL-Sim 固有の不具合ではなく、**コード署名のないインストーラー**でよく起きます。

【方法 A】一時的に緩和してからアンインストール
  設定 → プライバシーとセキュリティ → Windows のセキュリティ
    → アプリとブラウザーの制御 → スマート アプリ コントロール
  → 「オフ」または「評価」に変更（再インストール後は元に戻して構いません）
  → もう一度「HDL-Sim」のアンインストールを実行

【方法 B】フォルダを手動削除（ブロックを回避）
  1. HDL-Sim が起動中なら終了する
  2. エクスプローラーでインストール先フォルダを削除する
     例: C:\Users\<ユーザー名>\AppData\Local\Programs\HDL-Sim\
     （中身: HDL-Sim.exe, _internal\, unins000.exe など一式）
  3. スタートメニューの「HDL-Sim」ショートカットが残っていれば削除
  4. ドキュメント内のプロジェクトを残す場合は削除しない
     （data_dir.txt に書いた保存フォルダ、既定は ドキュメント\HDL-Sim）

  設定のアプリ一覧に「HDL-Sim」が残る場合:
  レジストリエディタ (regedit) で次のキーを削除（存在する方のみ）
    HKEY_CURRENT_USER\Software\Microsoft\Windows\CurrentVersion\Uninstall\
      {A1B2C3D4-E5F6-7890-ABCD-EF1234567890}_is1

0.5.5 以降の Setup で上書きインストールする場合
----------------------------------------------
旧版を完全に消さなくても、同じフォルダに新しい Setup を実行できることが多いです。
ただし _internal の不整合を避けるため、手動削除してから入れ直すのが確実です。

開発者向け（根本対策）
----------------------
Setup.exe と unins000.exe に Authenticode 署名を付けると、多くの環境で
ブロックされなくなります。手順は packaging/SIGNING.md を参照。
証明書は配布者が取得し、ビルド PC で HDL_SIM_SIGN_PFX 等を設定して
build_windows.bat / build_installer.bat を実行します。
